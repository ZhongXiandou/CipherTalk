# 首页“记忆客厅”设计规格

## 目标

在保留首页全屏背景、液态玻璃效果和“回忆一刻”核心体验的前提下，为宽屏首页增加明确的视觉层次与可用入口。首页应更像安静的个人记忆空间，而不是统计仪表盘或功能卡片集合。

成功标准：

- 第一屏同时具备用户身份、时间、核心回忆和最近联系四个层次。
- 背景仍是第一视觉资产，不被大面积面板遮挡。
- “回忆一刻”与最近联系人均可直接打开对应聊天。
- 数据库未连接、没有回忆或没有最近会话时，布局仍完整且不出现空洞卡片。
- 桌面窄窗口下内容不重叠，长用户名和联系人名称可安全截断。

## 设计方向

采用“记忆客厅”方案，不采用多卡片仪表盘。

```text
┌─────────────────────────────────────────────────────────┐
│ 晚上好，用户名                              开发者愿景   │
│ 7 月 20 日 · 星期一                                      │
│                                                         │
│            今天翻到的一段回忆                           │
│        [头像] [现有回忆气泡] [换一条]                    │
│                 来自 2022 年 7 月                        │
│                  打开这段对话                            │
│                                                         │
│ 最近联系                                                │
│ [头像 名称] [头像 名称] [头像 名称] [头像 名称]          │
└─────────────────────────────────────────────────────────┘
```

页面使用非对称布局：问候位于左上内容安全区，回忆主体保持视觉中心，最近联系人贴近底部内容安全区。右侧与背景主体区域保留负空间，供自定义背景和可拖动液态玻璃球展示。

## 视觉规则

- 不新增页面级白色卡片或卡片嵌套。
- 液态玻璃继续只用于回忆气泡和明确的交互控件，不把整块页面包进玻璃面板。
- 问候文字使用当前主题语义色；有背景媒体时保留现有高对比文字处理方式。
- 日期为次级信息，字号和对比度低于用户名。
- 最近联系人使用现有圆形头像视觉，最多展示 5 个；名称最多一行。
- 联系人项以头像为主，不使用大号矩形快捷入口。
- 动效仅使用 150-300ms 的淡入和轻微位移；遵守 `prefers-reduced-motion`。
- 不按视口宽度缩放字体，通过断点调整布局和间距。

## 组件划分

### `HomeGreeting`

职责：根据本地时间显示“早上好 / 下午好 / 晚上好”，显示当前用户昵称、月日和星期。

依赖：`useAppStore().userInfo` 与浏览器本地时间。不增加 IPC。

空状态：未加载到用户资料时使用“你好”；不显示“未连接用户”作为问候对象。

### `HomeMemoryFeature`

职责：封装现有“回忆一刻”标题、骨架、头像、消息气泡、刷新按钮，并补充消息日期与“打开这段对话”命令。

依赖：复用 `loadRandomMomentSnippet()`、`RandomMomentBubble` 和 `LiquidGlassBubble`。不改变随机回忆服务的抽样规则。

交互：

- “换一条”保持现有行为。
- “打开这段对话”调用扩展后的 `window.electronAPI.window.openChatWindow(sessionId)`。
- 打开失败时不破坏当前回忆，使用现有消息提示模式或可访问的内联状态。

### `RecentContactsRail`

职责：显示最近活跃的最多 5 个非系统会话，提供头像、名称和打开聊天能力。

数据：复用 `window.electronAPI.chat.getSessions(offset, limit)`，不直接在首页查询数据库表。新增 `src/features/home/recentContacts.ts` 作为纯函数模块，集中承载首页联系人资格、排序和截取规则，避免复制 `SessionSidebar` 的私有判断。

精确资格规则：

- 包含普通私聊、群聊（`@chatroom`）和企业微信联系人（`@openim`）。
- 排除空 username、`filehelper`、`gh_` 公众号、微信客服（`@kefu.openim`）、`service_` 服务账户。
- 排除 `isOfficialAccount`、`isOfficialFolder`、`isFoldGroup`、`isCollapsed` 标记的会话。
- 排除现有系统前缀：`weixin`、`qqmail`、`fmessage`、`medianote`、`floatbottle`、`newsapp`、`brandservicesessionholder`、`notifymessage`、`opencustomerservicemsg`、`notification_messages`、`userexperience_alarm`。

排序使用 `lastTimestamp || sortTimestamp` 降序，时间相同则按 `username` 升序保证稳定。每页读取 50 条，持续分页直到 `hasMore` 为 false 或累计扫描 200 条，再统一过滤、排序并截取前 5 个。后端分页使用 `sortTimestamp`，与首页最终排名字段不完全相同，因此不能找到 5 个合格联系人后提前停止。

交互：选择联系人时调用 `openChatWindow(username)` 打开独立聊天窗口并切换到目标会话。头像加载失败时使用名称首字 fallback。

空状态：无会话时整条轨道不渲染，不显示“暂无数据”卡片。

### `HomePage`

职责：保留背景媒体、开发者愿景弹窗和玻璃球，只负责编排三个首页内容组件。当前随机回忆状态可先保留在页面中，若拆分导致传参过多再下沉到 `HomeMemoryFeature`，不做无关状态重构。

## 数据流

```text
appStore.userInfo ───────────────> HomeGreeting

randomMomentService ─────────────> HomeMemoryFeature
                                      │ sessionId
                                      v
openChatWindow(sessionId)

chat.getSessions ──过滤/排序/截取──> RecentContactsRail
                                      │ username
                                      v
openChatWindow(username)
```

最近联系人只在数据库已连接时加载。使用 `useAppStore().myWxid` 作为账户作用域键；作用域变化时立即清空回忆、最近联系人和旧 `userInfo`，递增请求序号使上一账户的异步结果失效，然后为新账户重新加载三类数据。

账户切换成功后必须调用 `chat.getMyUserInfo()` 并以 `setUserInfo()` 替换 store 中的资料；获取失败则置为 `null`，不能继续显示上一账户昵称。随机回忆请求也必须把账户作用域键纳入依赖与 stale 判断，不能只依赖始终为 `true` 的 `isDbConnected`。

当前 `openChatWindow()` 只负责创建或聚焦窗口，不支持目标会话。本次需要沿用 `openMomentsWindow(filterUsername?)` 的既有模式扩展完整链路：

- `electron/preload.ts` 与 `src/types/electron.d.ts`：为 `openChatWindow` 增加可选 `sessionId`。
- `electron/main/ipc/windowHandlers.ts`：把可选 `sessionId` 传给窗口管理器。
- `electron/main/windows/windowManager.ts`：窗口已存在时发送 `chat:focusSession`；新窗口把目标会话放入启动查询参数，避免页面监听器尚未挂载时丢失事件。开发和生产加载路径都必须传递该参数。
- preload 暴露 `onFocusSession` 订阅接口。`ChatPage` 把启动参数或事件保存为 pending target，等待首批会话列表加载完成后查找目标会话；若首批不存在，则通过 `chat.getSessions(offset, limit)` 继续分页，直到找到目标或 `hasMore` 为 false。找到后将该 `ChatSession` 合并进 store，并调用现有 `handleSelectSession`，以同步完成当前会话、消息加载和缓存切换；不能只修改 `currentSessionId`。

不要依赖现有 `lastSession` 配置：当前聊天页没有消费该配置，而且它无法让已经打开的聊天窗口即时切换。

`openChatWindow(sessionId)` 的 Promise 只表示窗口已成功创建或聚焦、目标命令已派发，不承诺目标会话已经完成渲染。目标存在性由 `ChatPage` 在首批加载及必要的后续分页查找完成后校验：存在则消费 pending target 并调用 `handleSelectSession`；确认所有分页均不存在后才清除 pending target，并在聊天窗口显示一次“目标会话不存在”提示。后续更晚到达的会话刷新不会再次自动跳转。分页查找需带请求序号，较新的目标命令到达时使旧查找结果失效。

## 响应式布局

- `>= 1100px`：问候左上、回忆居中、最近联系人左下，使用完整名称。
- `760px-1099px`：三部分沿同一内容列排列，缩小纵向间距，最近联系人可减少为 4 个。
- `< 760px`：问候与回忆按文档流排列，最近联系人横向滚动；玻璃球不应遮挡主要交互。
- 所有固定区域使用明确的 `max-width`、`min-width: 0` 和安全边距；背景继续全屏裁切。

## 错误与加载状态

- 用户信息缺失：显示通用问候，不展示错误。
- 账户切换：立即清空上一账户问候、回忆和联系人；新账户数据未完成前使用通用问候与现有回忆骨架。
- 回忆加载：沿用当前骨架屏。
- 回忆为空：沿用当前简短占位提示，但不撑出大面积空面板。
- 最近会话加载：首屏不显示骨架轨道，加载完成后淡入，避免与回忆骨架竞争。
- 最近会话失败：静默隐藏轨道并记录控制台错误，不影响首页主体。
- 打开聊天失败：保留当前内容，向用户显示一次简短失败提示。

## 可访问性

- 首页问候使用 `h1`，回忆与最近联系使用有序标题层级。
- 联系人项与“打开这段对话”均为真实按钮，具备可读 `aria-label`。
- 头像图片包含联系人名称作为替代文本。
- 键盘焦点在复杂背景上仍有明确轮廓。
- 纯装饰背景与玻璃球保持正确的 `aria-hidden` 或无语义处理。

## 验证

- TypeScript 类型检查与 `git diff --check`。
- 使用已连接数据库分别验证：有回忆、有会话；无回忆；无最近会话。
- 验证点击回忆和最近联系人后，独立聊天窗口打开到正确会话。
- 分别验证聊天窗口未创建和已经打开两种情况下的目标会话切换。
- 验证账户切换后不残留上一账户联系人。
- 验证账户切换后不残留上一账户昵称或随机回忆，并覆盖旧异步请求晚于新请求完成的情况。
- 添加最近联系人纯函数测试，覆盖系统会话、公众号、折叠会话、群聊、企业微信、稳定排序与分页上限。
- 添加目标会话导航测试，覆盖事件早于会话列表、目标位于首批 300 条之外、目标不存在、快速连续目标、已打开窗口与新窗口启动参数。
- 检查浅色、深色及自定义图片/视频背景下的文字对比度。
- 检查宽屏、约 900px 窗口和最小支持窗口宽度，确认无重叠和文字溢出。
- 不执行应用构建，由开发者按项目约定自行构建。

## 不在本次范围

- AI 自动生成每日摘要或问候文案。
- 消息数量、联系人数量等统计仪表盘。
- 三条以上回忆轮播或时间轴。
- 修改随机回忆抽样算法。
- 新增首页背景或液态玻璃物理效果。
